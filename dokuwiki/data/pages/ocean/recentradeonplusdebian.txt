config: carte mere KT4V (AGP 3.0, aka 8x), Radeon 9600 pro (puce: RV350), le tout sous Debian testing/unstable

Le noyau ici est un noyau Debian binaire, kernel-image-2.6.4-1-k7. 

* installer les sources qui vont avec dans /usr/src/linux:
 su
 apt-get install kernel-source-2.6.4
 cd /usr/src
 tar xvjf kernel-source-2.6.4.tar.bz2
 rm -f linux
 ln -s kernel-source-2.6.4 linux
* telecharger ([URL au 27/03/2004 http://www.ati.com/support/drivers/linux/radeon-linux.html?type=linux&prodType=graphic∏=productsLINUXdriver&submit.x=8&submit.y=7&submit=GO%21]) le RPM du pilote proprietaire linux correspondant au serveur X (ici 4.2, la version se trouve au debut de /var/log/XFree86.0.log), par exemple le mien:
 fglrx-4.2.0-3.7.6.i386.rpm
* le transformer en .deb, et l'installer:
 alien fglrx-4.2.0-3.7.6.i386.rpm
 dpkg -i --force-overwrite fglrx*.deb
* maintenant il faut aller recompiler le module (remplacez 2.6.4-1-k7 par la version de votre noyau):
 cd /lib/modules/fglrx/build_mod
 ln -s /usr/src/linux /lib/modules/2.6.4-1-k7/build
 ln -s /boot/config-2.6.4-1-k7 /usr/src/linux/.config
* hacker un peu le script de compilation 'make.sh':
** si besoin (normalement pas besoin), vers la ligne 41, preciser le compilateur avec lequel votre noyau a ete compile (pour savoir le compilo du noyau en train de tourner : [=cat /proc/version]):
 CC=gcc-3.2
** vers la ligne 253, commenter le 'exit 1' juste au dessou de 'kernel includes ... not found or incomplete'
** vers la ligne 289, juste avant le 'if hit = 0 ... Error: kernel includes at ... do not match current kernel', ajouter une ligne 
 hit=1
* si vous avez une carte mere KT4V comme moi, il faut patcher le fichier agpgart_be.c: vers la ligne 6072, dans le bloc qui contient VIA_APOLLO_KT400, remplacer la ligne
 via_generic_setup }, // AGP v2
par
 via_kt400_setup }, // AGP v2
* compiler le module, et l'installer:
 sh make.sh
 cd ..
 sh make_install.sh
* generez un fichier de configuration pour XFree86 avec l'utilitaire de chez ATI :
 cp /etc/X11/XFree86-4 /etc/X11/XFree86-4.avant.fglrxconfig
 fglrxconfig
Demandez le support AGP du pilote ATI (pas celui du noyau). Dans le fichier genere, ca doit se concretiser par une ligne:
 Option "UseInternalAGPGART"         "yes"
Dites non a l'export des visuels en pseudocouleurs, ce qui doit donner la ligne:
    Option "PseudoColorVisuals"         "off"
(en mettant ca a "on", mon extension xvideo, qui permet de voir des videos avec acceleration materielle, ne marchait plus)
Faites ce que vous voulez avec l'antialiasing (AA). Je l'ai activé, et j'ai laissé les options par défaut (en particulier, je ne force pas l'AA sur toutes les applis). Sinon, je tourne en Single Head, et  j'utilise le mode TLS "safe" (2). Activez la synchronisation sur l'affichage, c'est beaucoup plus joli, et ça ne sert à rien de faire 4500 frames/s de toute façon.
* relancer X, par exemple si vous avez wdm (a faire dans la console, pas sous X):
 /etc/init.d/wdm restart

Pour tester:
 fgl_glxgears # montre un cube qui tourne avec des engrenages animés sur toutes les faces
(il est *normal* d'avoir 75 f/s si vous activé la synchro sur le balayage vertical -- sinon vous devez avoir plusieurs milliers de f/s sur une carte moderne)
Version classique:
 glxgears
Capacités d'affichage:
 glxinfo # si ça parle de Mesa, c'est pas bon, sinon, ça va
Extension XVideo, nécessaire pour lire des vidéos dans de bonnes conditions:
 xvinfo # ne doit pas donner 'no adapters'
Et enfin:
 tuxracer
doit être jouable de manière fluide. Même sur mon Athlon 2600+ le menu principal rame en rendu soft.
