=== un exemple ===

Supposons que je veuille faire partager au monde un tas de code utile, qui permet de gerer des vecteurs dans l'espace 3d. Je me fais un repertoire comme ceci:

 vector3d
 |-- README
 |-- Makefile
 |-- Doxyfile
 |-- include
 |   |-- utilitaires_vector3d.h
 |   `-- vector3d.h
 |-- src
 |   |-- utilitaires_vector3d.cpp
 |   `-- vector3d.cpp
 |-- lib
 |   `-- libvector3d.so
 `-- doc


=== que distribuer ? ===

Ce qui est interessant pour la reutilisation par quelqu'un d'autre, c'est
0 une librairie, libvector3d.so ou libvector3d.a
0 les fichiers .h qui vont avec (dans leur repertoire 'include')
0 le README

Du code correctement encapsule devrait tenir dans une librairie, et l'utilisateur ne devrait pas avoir a le modifier. Les fonctionnalites fournies devraient etre coherentes (pas question de packager Aladyn dans une seule grosse bibliotheque, par exemple).

=== les fichiers ===
==== le repertoire de base : vector3d
Tous les fichiers ci-dessous doivent se trouver dans un repertoire qui porte le nom du module.
==== README
Il s'agit de decrire brievement ce que fait la librairie, comment la compiler, comment l'utiliser.
==== Makefile
Il doit y avoir une cible qui permet de construire dans 'lib/' la librairie libvector3d.so ou libvector3d.a, et la documentation doxygen du projet.
==== Doxyfile
Ce fichier doit permettre de generer la doc doxygen du projet dans doc/, a partir des sources (.c/cpp et .h), au format HTML. Un fichier Doxyfile generique qui peut marcher pour tout est donne dans AGenericDoxyfile.
==== include
Le repertoire 'include/' doit contenir les fichiers .h necessaires a l'utilisation de la librairie. Leur liste doit etre donnee dans le README. (Si les fichiers .h sont places ailleurs dans l'arborescence du projet, il suffit de faire des liens symboliques relatifs dans 'include/'.)
==== src
Si possible, les sources devraient etre regroupees dans un repertoire 'src/', mais ca n'est pas obligatoire.
==== lib
Ce repertoire doit contenir la librairie associee au projet, libvector3d.so ou libvector3d.a, telle qu'elle est construite par le Makefile.
==== doc
Ce repertoire doit recevoir la doc doxygen du projet.
