Cette page parle de restaurer des fichiers stockés sur andrinople, à partir des sauvegardes automatiques qui sont faites toutes les nuits. Pour restaurer des fichiers sur une partition ext2, dont on n'a pas de backup récent, voir RestoringDeletedFilesFromAnExt2Partition.

J'ai effacé mon répertoire .mozilla de mon compte, hébergé sur andrinople (comme tout CyberMove). Voici comment je l'ai restauré. % est l'invite du shell, # indique un commentaire, > est l'invite de recover, l'utilitaire de restauration (bah, tant pis pour l'anglicisme). Cette procédure doit être valable pour tous les fichiers à restaurer qui sont stockés sur votre home sur andrinople.

  % telnet andrinople
  # se logger normalement

  % cd /local_home/lehy
  % recover -s back-serv

  # add: marquer des fichiers/r�pertoires pour restauration
  > add .mozilla

  # lancer la sauvegarde, par défaut la dernière disponible
  # ça peut prendre un bon moment, même pour quelques petits fichiers
  > recover

Pour voir les diff�rentes versions du fichier qu'on veut restaurer:
  versions .mozilla
En l'occurence, j'ai pu remettre une sauvegarde de 2 jours plus tôt (avant le we, donc tout va bien).

On peut aussi se déplacer dans l'arborescence avec :
  > cd
  # et
  > ls

Et avoir de l'aide sur les commandes avec :
  > help

Enfin, on peut voir à ce sujet les pages des MI sur l'intranet, mais elles ne sont pas terribles-terribles, je trouve. A vous de juger:
http://www.inrialpes.fr/irami/local/systeme/sauvegarde_user.html
