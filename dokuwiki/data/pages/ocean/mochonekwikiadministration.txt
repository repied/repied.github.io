This kwiki runs kwiki 0.18, installed from a Debian package (originally 0.17, then upgraded).

[A few patches have been applied http://www.kwiki.org/index.cgi?KwikiDevelBugs]. The problem of page name under mod_perl has also been fixed (in Kwiki::handler, set $0 appropriately).

There's a custom formatter (RonansFormatter.pm) that inherits from CGI::Kwiki::Formatter, and handles my extensions, like links to ProBT docs, or relative http:// links.

There's a custom template (RonansTemplate.pm) that inherits from CGI::Kwiki::Template, and adds a link to AuSujetDuLogo on the logo.

I use a customised Plasma CSS (Plasma_lighter.css).

I use CGI::Kwiki::Backup::SVN as a version control solution (which caused me to install libsvn0, libxml2, and libassomething from unstable). I used to use RCS, but it caused problems, and didn't work very well.

Now with SVN history and locking seem to work fine, although the Apache error log shows repeated 'child died (segfault)'.

I patched dirtily Kwiki.pm, by changing
  $ADMIN ||= 0;
to
  $ADMIN ||= 1;
in order to have admin access work. As a result, everybody is granted admin access. I'd like to have a clean solution to this instead.
