Je veux déboguer mon programme avec gdb, dans emacs. Mon programme s'appelle 'visualise', et il prend un argument, qui est 'alpages.OpenMASK3'; pour le lancer normalement, je fais:
 ./visualise alpages.OpenMASK3

Je le compile avec les flags suivants:
 -g -gdwarf-2 -g3
Dwarf-2 est le format d'infos de débogage, il semble que ça soit ce que gdb aime le mieux ces temps-ci. -g3 dit qu'il faut inclure les infos de débogage sur les macros.

Dans emacs:
 M-x gdb
Puis:
 Run gdb (like this): gdb visualise

Ensuite c'est l'invite gdb:

 (gdb) run alpages.OpenMASK3

Quand mon programme plante, j'affiche la pile des appels:
 (gdb) bt

Pour naviguer dans la pile:
 C-< pour remonter, ou la commande gdb 'up'
 C-> pour descendre, ou la commande gdb 'down'

Il y a un menu 'gud' avec les commandes les plus utiles, et les raccourcis, si on les oublie.

CedricPradalier a trouvé une façon d'appeler operator<< pour imprimer un objet. Il suffit de le wrapper dans une petite fonction C, qu'on insère dans son programme. Par exemple, pour imprimer n'importe quel objet PL (sauf plData, qui peut être fait sur le même modèle):
 using namespace std;
 void ppl(void * o)
 {
     // On change de type
     plOpenplObj * O = (plOpenplObj*)o;
     (operator<<(cout,*O)) << endl;
 }

Pas de moyen connu d'accéder directement à cout, cerr, et autres membres de std:: à partir de gdb (ou alors gdb ne connaît pas leur type).
