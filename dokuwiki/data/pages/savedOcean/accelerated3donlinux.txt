Deja c'est quoi les versions du serveur X et
du noyau? Aux dernieres nouvelles (pas verifie), la derniere version de
X dans testing, c'est la 4.2.1.  Un peu ancienne, mais d'apres mon
experience c'est celle qui marche le mieux avec les radeons VE AGP sur
x86. Et pour le noyau, c'est je crois le 2.4.20

Y'a 2 parties pour le support 3D dans linux : 1 module du noyau (DRM),
et la partie X.  Il faut que les 2 aillent ensemble. Autrement dit, si
tu prends 1 X + recent, il faudra probablement aussi 1 module DRM autre
que celui qui est dans le noyaux standard...

Ca peut se faire, et y'a meme 1 procedure debianisee pour ca. Moi j'y ai
ete oblige, car il faut X 4.3 pour ma radeon8500.  Y'a 1 inconvenient de
taille : crash de ma machine a la fermeture de la session X si j'utilise
1 gestionnaire de session (xdm, wdm, gdm...).  Donc j'ai vire ce
dernier, et je suis oblige de me loguer sur la console texte et faire
"startx". C'est toi qui voit 

Toujours motive? Bon, ajoute la ligne suivante dans
 /etc/apt/sources.list

 deb http://penguinppc.org/~daniels/sid/i386 ./

 apt-get update && apt-get upgrade && \
 apt-get install xlibmesa4-dri xlibmesa4-glu

Avec de la chance ca suffira. Relance X pour voir:
 /etc/init.d/wdm restart 
Sinon, il faut
s'occuper du module DRM.  Helas oui, ca veut dire recompiler 1 noyau (il y a plus de détails sur la compilation d'un noyau à partir des sources Debian dans CompilingAKernelTheDebianWay).

 apt-get install xlibmesa4-drm-src kernel-source-2.4.20
 cd /usr/src
 tar xjf kernel-source-2.4.20.tar.bz2
 rm -f linux && ln -s kernel-source-2.4.20 linux
 tar xzf xlibmesa4-drm.tar.gz
 cd linux && make-kpkg clean && make menuconfig

Choisis "Load an Alternate Configuration File" (a la fin du menu) pour
recuperer ta config actuelle. Le chemin est 1 truc du genre :
 /boot/config-2.4.20  
(uname -a pour connaitre ta version courante).
Puis dans le menu "character devices", selectionne "direct rendering
manager" mais _deselectionne_ tous les drivers DRM en dessous.
exit, save.

 make-kpkg --initrd --revision=ronan.1 kernel_image modules_image
 cd .. && dpkg -i kernel-image-2.4.20_ronan.1_i386.deb
 dpkg -i xlibmesa4-drm-module-2.4.20_ronan.1_i386.deb

et reboote!
Vivement le jour ou ce sera + simple d'upgrader les drivers sous
linux...
