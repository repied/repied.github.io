[pbuilder http://www.netfort.gr.jp/~dancer/software/pbuilder-doc/pbuilder-doc.html] is used to create a chroot (contained in a .tar.gz file), that you can use to test software on. I used it to create a 'sarge' environment, to test whether PyPL compiles and works there (my machine has drifted away from a sarge, and I have patched various things -- that's why I want to test it in a clean sarge install).

 % pbuilder create --distribution sarge
 
 % pbuilder update --distribution sarge --extrapackages 'swig1.3 python python-tk python-dev xlibmesa3 scons jed'
 
 % cat install_zinc
 #!/bin/sh
 dpkg -i /scratch2/downloads/zinc-tk_3.2.100_i386.deb /scratch2/downloads/zinc-python_3.2.100_all.deb
  
 # does not work, pbuilder cleans up afterwards, so things don't remain installed
 % pbuilder execute --distribution sarge --bindmounts /scratch2 install_zinc
 
 % pbuilder login --distribution sarge --bindmounts /scratch2
