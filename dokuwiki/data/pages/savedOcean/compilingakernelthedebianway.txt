Pour juste installer un noyau Debian sans le recompiler, voir InstallingAStockDebianKernel.

Je veux recompiler un noyau 2.6.0, par exemple.
 apt-get install kernel-source-2.6.0
 cd /usr/src
 tar xjf kernel-source-2.6.0.tar.bz2
 rm -f linux && ln -s kernel-source-2.6.0 linux
 cd linux
 make-kpkg clean
 make menuconfig
A ce point, il faut configurer les différentes options de compilation du noyau. Il est possible cependant (tout en bas dans le menu) de charger une configuration existante, qu'on a édité et sauvé à la main, ou venue avec un noyau Debian tout packagé. Les configurations sont stockées dans /boot, et la config de mon noyau Debian packagé, 2.6.0 version 1 pour AMD Athlon (aka K7) (apt-get install kernel-image-2.6.0-1-k7) (pas compilé par moi) s'appelle /boot/config-2.6.0-1-k7. La configuration du dernier noyau compilé est /boot/.config, mais je ne suis pas sûr de ça.
Ensuite, on peut lancer la compilation :
  make-kpkg --initrd modules-image kernel-image
Il va y avoir un avertissement pour savoir si on veut vraiment créer un initrd. Tout va bien, c'est ce qu'on veut (les sources Debian du noyau sont prévues pour ça).

Enfin, installer le noyau (jusqu'ici on n'a rien cassé, à part /usr/src/linux qui a été modifié) :
  cd /usr/src
  dpkg -i kernel-image-2.6.0-1-Custom # ou un truc du genre, il y a Custom dedans en tous cas

Ne pas oublier de metter à jour son bootloader (lilo ou grub en général). Un conseil: jetez lilo, installez grub (voir UsingGrub), ça vous évitera beaucoup de prises de tête.
Si on a lilo quand même, le dpkg ci-dessus propose de mettre à jour sa config pour pouvoir lancer le nouveau noyau. Moi j'ai grub, donc je dis 'non' quand il me propose, et je fais juste :
  update-grub
qui ajoute gentiment une entrée dans /boot/grub/menu.lst pour que grub puisse lancer mon noyau.

=== notes ===
* L'idée est d'avoir toujours dans /usr/src un répertoire 'linux', qui est en fait un lien symbolique sur les sources du noyau qu'on utilise (ça n'est pas strictement indispensable quand la machine tourne, mais ça peut probablement éviter pas mal de maux de tête).
* Ne pas oublier le flag --initrd à make-kpkg, sinon le noyau risque de ne pas être bootable.
* Les paquets Debian des sources du noyau incorporent des patches propres à Debian (qu'on ne trouve pas dans les sources du noyau telles qu'on peut les télécharger sur http://www.kernel.org).
* Pour connaître la version du noyau qui tourne sur sa machine :
  uname -a
