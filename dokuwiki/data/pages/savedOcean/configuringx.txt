= XF86Config-4 =

Pour utiliser des polices True Type (vectorielles, se pretant bien a l'antialiasing), il faut declarer le serveur de police idoine (xfs-xtt, qui est generalement installe, sinon faut l'installer) dans son /etc/X11/XF86Config-4, dans la section "Files":
    FontPath        "unix/:7110"                    # local font server (xfs-xtt)

Si on fait cette modification, il faut redemarrer le serveur X pour qu'elle soit effective.

De bonne polices TrueType libres sont celles de la famille Bitstream:
 apt-get install ttf-bitstream-vera

= gestionnaires de login =
Il existe plusieurs gestionnaires de login (programmes qui permettent de gérer le login, et le lancement  du serveur X avec le bon environnement graphique (gestionnaire de fenêtres (WindowMaker, Metacity, Enlightenment, Sawfish...) et éventuellement gestionnaire de bureau (Gnome, KDE, GNUStep...)). Les plus courants: xdm, gdm, wdm...

Pour l'installation, rien de bien compliqué:
  apt-get install xdm # ou gdm ou wdm

Par défaut avec wdm, un autre utilisateur (comme root) que celui qui a lancé X ne peut pas afficher des applications sur le bureau, ce qui donne des messages comme:
 ocean:/etc/X11/wdm# xclock
 Xlib: connection to ":0.0" refused by server
 Xlib: No protocol specified 
 
 Error: Can't open display: :0.0

Pour autoriser ça, l'utilisateur qui a lancé X doit faire dans son X (et *en local sur la machine où tourne le serveur*):
 xhost +grenat
pour autoriser que des connections depuis grenat affichent des trucs sur le serveur X,
 xhost +localhost
pour autoriser n'importe qui connecté en local (même via ssh ou telnet) à afficher des trucs sur le bureau, et
 xhost +
pour autoriser *n'importe qui de n'importe où* à afficher n'importe quoi sur le serveur (ce qui permet de le locker par exemple, avec xlock, ou de faire des tas de choses plus ou moins drôles).

Il semble que par défaut (en Debian en tous cas) gdm n'active pas le contrôle d'accès.

Si on veut systématiquement désactiver le contrôle d'accès pour la machine locale à chaque fois qu'on se connecte dans une session graphique, on peut mettre
 xhost +localhost
dans un fichier nommé [=~/.xsession] ; tout ce qui est dedans est exécuté quand la session démarre.

Il existe un mécanisme plus compliqué de sécurité, avec une histoire de cookies MIT, de XAuthority, et de choses comme ça, mais personne ne sait comment ça marche.
