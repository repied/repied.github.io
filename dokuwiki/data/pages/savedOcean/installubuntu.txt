= Réseau
Lors de l install (ou apres) il faut donner le bon nom (turquoise pour moi) et la bonne IP (194.199.21.200, masque 255.255.255.0), ainsi que la passerelle 194.199.21.90 et les dns de l inria 194.199.15.65.

= Nis:
C'est ce qui permet de se loggeur sur une machine en atant qu utilisateur deporte (yellowpages)
Il faut declarer les seveurs nis et le domaine, ca qui ce fait dans le fichier 
 /etc/yp.conf

 domain raminis server nis1-serv.inrialpes.fr
 ypserver nis2-serv.inrialpes.fr
 ypserver nis3-serv.inrialpes.fr

Il faut aussi mettre le bon ordre dans 
 /etc/nsswitch.conf
(voir fichier) Il faut donc installer les nis, puis redemarer le serveur 
 sudo invoke-rc.d nis restart

= Nfs
Le minimum est de montee le home de emotion. Ca ce fait avec autofs. Voir mes fichiers
 /etc/auto.master 
qui apelle
 /etc/auto.mp3

Attention les fichietrs du type /etc/auto.mp3 ne doivent *pas* avoir les droits en ecriture. Sinon, ca plante et ca ne dit pas pourquoi. Avec autofs, on ne voit les repertoire qu en faisant un cd explixite dans eux, sinon, on a l impression qu il n y a rien.

Sinon, on peut monter des trucs en dur dans le fichier /etc/fstab, mais la y aurra des pb si la machine pointee nest pas disponible. Au final je n ai pas touche a ce fichier.

Un pb a faire gaffe est que tout ce qu il y avait avant dans le /home sera recouvert, par exemple le home de l utilisateur local. C est pourquoi il faut le deplacer et monter les autres trucs ailleurs. pour cela il faut demonter le /home de emotion, deplacer le home de localdangauth pour le mettre par ex dans /local_home, modifier aussi ceci dans le /etc/passwd. Attention il faut pas toucher direct ce fichier m ais passer par
 EDITO=emacs vipw
Pour avoir dansle le etc/passwd
 localdangauth:x:1000:1000:localdangauth,,,:/local_home/localdangauth:/bin/bash

Pour le redemarer
 sudo invoke-rc.d autofs restart
Attention desfois le stopper ne marche pas vraiment, voir Ronan Lehy pour ces subtilités subtiles.

= Compte root
Par default, on ne peux pas se logguer en tant que root sous les ubuntu, et le mot de passe root est celui de l utilisateur lmocal. Ca se discute, masi en tout cas c e n est aops coherent avec la politique de gestion des machines de l'équipe, pour lesquels les mots de passe root doivent etre standart. Il faut donc autoriser root a se connecter avec le mot de passe standart que je ne revelerai pas ici. Pour cela il faut bidouiller le fichier des mots de passe.

= Son
Par default et par souci de securite,les nouveaux utlisateurs n on pas droit de jouer du son ni de de video ni de cdrom (trop cool). Et bien sur le message d erreur est super explixit, du style "pas de carte son installee". En fait il faut ajouter les utilisateurs aux groupes necessaires:
 sudo addgroup dangauth audio
 sudo addgroup dangauth video
 sudo addgroup dangauth cdrom

= Driver nvidia
Mettre les bons paquets par l interface des paquest Ubuntu :
 sudo apt-get install nvidia-glx nvidia-kernel-common
 sudo nvidia-glx-config enable

Puis redemarerr le serveur X.

Puis aussi il va falloir aller bnidouiller dans le /etc/X11/xorg.conf pour avoir de bonne resoltuion et frequence.
