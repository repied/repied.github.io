Maple 9 can be used under Linux. It resides on soft-serv.inrialpes.fr. I have here a few very basic notes about it.

=== where is Maple? ===
On my Debian machine, soft-serv is mounted in /softs, so I just added /softs/bin to my path. If you have a Red Hat, write here where /softs is mounted.

=== running Maple ===
So just type 
  xmaple
and voilà. Hours of fun with Java eating all the RAM it can find just to display aliased equations, and taking centuries to save a worksheet or display a button. I hear some use Maple without the graphical interface, that is just
  maple

*Important note*: to run, xmaple needs a library named [=libstdc++-libc6.1-1.so.2] (if it's not there, [=xmaple] does nothing, just returns without any output). In Debian, it used to be found in a package named [=libstdc++2.9-glibc2.1], which is no more in the official archive. You may try this dirty hack (which works for me):
 su
 aptitude install libstdc++2.10-glibc2.2
 ln -s /usr/lib/libstdc++-libc6.2-2.so.3 /usr/lib/libstdc++-libc6.1-1.so.2 

=== what statements display ===
A statement with a colon (:) at the end shows no output, as in
  formula := x * y:
A statement with a semicolon shows output, which can be enough to bring your P4 war machine to its knees :
  formula := x * y; # not that one, though

=== executing a complete worksheet ===
The !!! button executes the full worksheet.

=== generating C code ===
You can generate 'optimised' C code for a formula like this:
  formula := x * y * exp(sin(Pi*theta))
  with(CodeGeneration);
  C(formula, optimize=true);

=== animations ===
You can animate your graphs in Maple.
First, load the plots package:
  with(plots):
Then for a 2d animation you can type:
  animate(plot,[f(x,t),x=xmin..xmax],t=tmin..tmax);
For surfaces, it involves [=plot3d]:
  animate(plot3d,[z(x,y,t),x=xmin..xmax,y=ymin..ymax],t=tmin..tmax);
You can also feed a vector for a parametric plot:
  animate(plot3d,[[x(u,v,t),y(u,v,t),z(u,v,t)],u=umin..umax,v=vmin..vmax],t=tmin..tmax);
More generally, the first argument is a plot command, followed by a vector of its arguments (including optionally the plot options), then the range of the animation parameters then finally the options of the animation.
For a real example:
  animate(plot3d, [[x,y*cos(t)-x*sin(t),x*cos(t)+y*sin(t)],x=-1..1,y=-1..1,orientation=[90,0],projection=0.9,style=WIREFRAME,color=black,view=[-1.5..1.5,-1.5..1.5,-10..10]], t=0..1,paraminfo=false,frames=40);
If you want to save your animation, change the [=plotsetup] just like a static graph:
  plotsetup();    % display actual setup
  plotsetup(gif,plotoutput="/home/.../myfile.gif");    % change device and file of plot commands
  plotsetup(default);    % back to defaults
  plotsetup(x11);    % graphical mode (if default device is terminal)

=== various useful functions ===
* to get the indeterminate symbols in an expression:
  indets(expr, symbol);
* to clean the environment ("on tue tout le monde et on repart sur des bases saines", Jean Rochefort, /Cible Emouvante/ (Pierre Salvadori)):
  restart;
* to erase any numeric value given to a symbol, and restore it as a symbol:
  alpha := 'alpha';
* the 'sign' function:
  s := signum(arctan(x));
* a comment
  % this is a comment
